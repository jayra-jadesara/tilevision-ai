feat: Tasks A-E - Professional Dashboard, Duplicate Detection UI, Search UX, Settings, Help/User Guide

TASK A - Professional Dashboard:
Rebuilt DashboardView with real stat cards backed by live data instead of
the previous 2-card version: Total Images, Indexed Folders, Database
Size, FAISS Index Size, Last Search, License Status, Trial Remaining,
Watched Folders. Added Recent Activity and Recent Searches panels (click
a past search to re-run it). Quick Actions now cover all four requested
targets: Search, Index Folder, Duplicate Detection, Settings (previously
only Index/Search existed).

New data layer to support this: search_history and activity_log SQLite
tables, ISearchHistoryRepository/IActivityLogRepository +
SQLite implementations. SearchViewModel and IndexingViewModel optionally
record into these (kept optional so existing construction sites/tests
without them keep working). Added DashboardDataProviders, a small bundle
class, rather than growing MainWindow's constructor with 6+ more
individual callables.

TASK B - Duplicate Detection UI:
Each duplicate tile card now shows a Duplicate % badge (100% for exact
matches; a Hamming-distance-derived percentage for near-duplicates,
computed against the group's reference tile) and an Exact/Near badge.
Added a real Delete action: confirms with the user, then removes the
file from disk, its FAISS vector, and its SQLite row via a new
FindDuplicatesUseCase.delete_duplicate() method. Groups automatically
collapse once fewer than 2 files remain (no longer a "duplicate").

TASK C - Search UX:
- Elapsed search time + results count shown after every search
  (search_stats_ready signal).
- Animated "Searching..." dots while a search is in flight.
- Empty state now has a larger icon alongside the text (illustration).
- Large Preview: single-clicking a result opens a reusable preview panel
  with a bigger thumbnail + full metadata (double-click still opens the
  image in the OS viewer, unchanged).
- Search History: a "Recent Searches" button showing a popup menu of past
  searches, click to re-run one (SearchViewModel.repeat_search()).
- Right-click menu gained "Copy Path" alongside the existing Open
  Image/Open Folder actions.

TASK D - Settings:
- Backup Database (copies the SQLite file to a chosen location).
- Rebuild FAISS Index: force-reembeds every previously-indexed folder via
  a new RebuildIndexWorker (background QThread + progress dialog). Added
  a `force` parameter to scan_and_index_directory() that bypasses the
  unchanged-file hash check — the only way to truly rebuild the index
  given that raw embedding vectors aren't persisted anywhere except
  inside the FAISS file itself.
- Clear Cache (deletes cached thumbnails, regenerated on next view).
- Export Logs (copies the current log file to a chosen location); added
  get_log_file_path() to utils/logger.py.
- Thumbnail Size setting (new AppSettings.thumbnail_size property).
- Database Path display.
- Language dropdown placeholder (English only, disabled, "coming soon"
  tooltip) per the spec's "Language (future)" item.
- Cleaned up 5 redundant explicit settings.save() calls discovered while
  making these changes — every AppSettings property setter already
  auto-saves internally, so the explicit calls were dead weight (Task 12:
  no duplicate/dead code).

TASK E - Help / User Guide:
New HelpView modal dialog: the 5-step walkthrough (Choose a Folder ->
Index Images -> Upload a Customer Image -> View Similar Tiles ->
Double-Click to Open), each with a screenshot placeholder that
automatically renders a real image instead if one is later dropped into
src/resources/help/ (no code changes needed for that swap). Opened via a
new "Help" sidebar button.
