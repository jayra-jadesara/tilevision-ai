fix: address user feedback - search results dropdown, theme system, branding, help clarity

Fixes reported from real usage testing (screenshots), addressing each
point directly:

1. "Search result show in dropdown 5,10,15,20,25"
   Settings' "Search Results Shown" was a free-form QSpinBox (1-100).
   Replaced with a QComboBox of exactly those five presets. If an existing
   saved value isn't one of them (e.g. from before this change), it's
   inserted and shown rather than silently changed out from under the user.

2. "Thumbnail size automatic - no need to show there"
   Removed the Thumbnail Size control from Settings entirely. The backend
   AppSettings.thumbnail_size property is left in place (harmless,
   avoids touching unrelated code) but nothing in the UI exposes it
   anymore.

3. "Theme not worked properly"
   This was real: every view hardcoded its own dark-only colors directly
   in a local setStyleSheet() call, so switching themes only ever
   re-skinned the MainWindow chrome + shared combo/menu/table/scrollbar
   fixes - each view's own panels, cards, and text stayed dark regardless
   of the selected theme, producing a broken half-applied look.

   Fixed properly: added a shared color palette (theme/theme_manager.py's
   get_palette()) with matching keys across both themes, and every view
   (Indexing, Search + its preview panel, Dashboard, Settings, Duplicates,
   Crop, Help) now builds its stylesheet from that palette instead of
   hardcoded hex, and exposes set_theme() to rebuild it on demand.
   MainWindow now propagates a theme change to every currently-open view
   immediately, and passes the current theme to any dialog constructed
   afterward (Duplicates, Crop, Help) so newly-opened ones match too.
   Also found and fixed a real pre-existing bug while doing this: the
   Modified/Deleted stat value labels on the Index page (added with Task
   2) had no styling at all in the old stylesheet - they'd have rendered
   in default black/unstyled text, effectively invisible on the dark
   background.

4. "Thumbnail & DB path no need to show client...may accidentally delete"
   Removed the Database Path and Thumbnail Cache rows from Settings'
   Overview section. The database path is still used internally for the
   Backup Database action - just never displayed as raw text a client
   could navigate to and delete.

5. "TileVision AI need a logo"
   Generated a real logo (rounded indigo badge, 2x2 tile grid with a
   magnifying glass over one tile - the visual-search motif) via PIL, at
   src/resources/logo.png (512px), logo_small.png (96px, sidebar), and
   app_icon.ico (multi-resolution Windows icon). Wired into: the sidebar
   (replacing the plain "TV AI" text), the window titlebar icon, and the
   QApplication-level icon (Windows taskbar).

6. "Project made by JD Software, add contact number 88662 77767"
   Set QApplication's organization name to "JD Software". Added a
   branded footer to the Help dialog (logo + "made by JD Software" +
   contact number). Also added the same credit as a tooltip on the
   sidebar's version label (v1.0.0), which is now clickable and opens
   Help directly - the sidebar itself is too narrow (104px) to fit the
   full credit line as visible text.

7. "Help Popup more user friendly need, more difficult to understand"
   Rewrote HelpView: each step is now one short, plain-language sentence
   instead of a paragraph, with a single icon-badge per step instead of a
   plain number, and a friendlier heading ("How TileVision AI Works").
